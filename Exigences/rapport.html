Acheteurs
Désigne ceux qui magasinent, achètent et évaluent les produits sur UniShop

Description des cas d'utilisation

Cas d'utilisation : S'inscrire comme acheteur.
But : Permettre à un nouvel utilisateur de créer un compte acheteur.
Préconditions : L'utilisateur est sur la page d'inscription.
Acteurs : Nouvel utilisateur (principal), Système (secondaire).
Scénario principal :
1. L'utilisateur accède à la page d'inscription.
2. L'utilisateur entre ses informations de profil : nom, prénom, pseudo (unique), adresse courriel, téléphone.
3. L'utilisateur entre son adresse d'expédition.
4. L'utilisateur soumet le formulaire d'inscription.
5. Le système valide les informations et crée le nouveau compte.
Scénarios alternatifs :
5a.1. Si les informations fournies ne sont pas valides, le système affiche un message d'erreur et invite l'utilisateur à corriger les informations.
Postconditions : Un nouveau compte acheteur est créé. L'utilisateur peut maintenant se connecter à UniShop avec ce compte.

Cas d'utilisation : Modification du profil.
But : Permettre à un membre existant de modifier son profil.
Préconditions : Le membre est connecté à son compte sur UniShop.
Acteurs : Membre (principal), Système (secondaire).
Scénario principal :
1. Le membre accède à la page de son profil.
2. Le membre modifie les informations de son profil.
3. Le membre soumet le formulaire de modification.
4. Le système valide les modifications et met à jour le profil du membre.
Scénarios alternatifs :
4a.1. Si les modifications ne sont pas valides, le système affiche un message d'erreur et invite le membre à corriger les informations.
Postconditions : Le profil du membre est mis à jour avec les nouvelles informations.

Cas d'utilisation : Afficher une liste de produits disponibles.
But : Permettre à un utilisateur de voir tous les produits disponibles à l'achat.
Préconditions : L'utilisateur est sur la page des produits de UniShop.
Acteurs : Utilisateur (principal), Système (secondaire).
Scénario principal :
1. L'utilisateur accède à la page des produits.
2. Le système affiche une liste de tous les produits disponibles.
Scénarios alternatifs :
2a.1. Si un produit est indisponible, le système affiche un message pour ce produit indiquant qu'il n'est pas actuellement disponible.
Postconditions : L'utilisateur peut voir tous les produits disponibles à l'achat et peut sélectionner un produit pour plus de détails ou pour l'achat. Si un produit est indisponible, un message est affiché pour ce produit.

Cas d'utilisation : Aimer des produits.
But : Permettre aux acheteurs de "liker" les produits pour les faire découvrir plus facilement par d'autres acheteurs potentiels.
Préconditions : L'acheteur doit être connecté à UniShop.
Acteurs : Acheteur (principal), Système (secondaire).
Scénario principal :
1. L'acheteur navigue à travers les produits.
2. L'acheteur sélectionne un produit qui lui plaît.
3. L'acheteur clique sur le bouton "like" du produit.
4. Le système enregistre le "like" de l'acheteur.
Scénarios alternatifs :
3a.1. Si l'acheteur a déjà "liké" le produit, le système retire le "like".
Postconditions : Le "like" de l'acheteur est enregistré et visible par d'autres utilisateurs.

Cas d'utilisation : Recherche de produits.
But : Permettre aux utilisateurs d'effectuer des recherches par mot-clé pour trouver des produits spécifiques.
Préconditions : L'utilisateur doit être connecté à UniShop.
Acteurs : Utilisateur (principal), Système (secondaire).
Scénario principal :
1. L'utilisateur entre un mot-clé dans la barre de recherche.
2. L'utilisateur sélectionne les filtres souhaités.
3. L'utilisateur lance la recherche.
4. Le système affiche les produits correspondant aux critères de recherche de l'utilisateur.
Scénarios alternatifs :
4a.1. Si aucun produit ne correspond aux critères de recherche, le système affiche un message indiquant qu'aucun produit n'a été trouvé.
Postconditions : Les produits correspondant aux critères de recherche de l'utilisateur sont affichés.
